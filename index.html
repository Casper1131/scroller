This plugin can add a horizontal scroll ui in the form of buttons, one to scroll left and the other to scroll right, to any "unordered" list,this can also be implemented for jquery ui tabs and more

Options:

proportionThe size of slidable area as a decimal or percentage.NB pixels can be used but it would be converted to a decimal that is proportional to parent size.
SpaceThe space between button and slider area,defualt is auto in which plugin will determine the space based on padding and margin values
selectionThe selector string or dom object of the current selected li.NB The plugin does not assign any id or class to li's on any trigger the user will have to define those functions
htmlThe html to be added to left and right button.example html:{buttonLeft:'left',buttonRight:'right'}
bspaceThe initial space between buttons and list.example bspace:10. The (integer)space is converted to a percentage of parent width.
backgroundFor UI implementations a background is appended to parent dom, this background copies the background properties of the list. If you desire to change the properties of this background it can be done by setting the css and position of background.NB both must be set also there only exist one position option "absoluteTop". example background:{position:'',css:{background:'red'}} or :background{position:'absoluteTop',css:{background:'red'}}

Methods:

resetReset position of ul to left 0
getLastGet last visible item still in view.
getFirstGet first visible item still in view.
isOverBoolean is there an element beyond the view of scroller
isUnderBoolean is there an element below the view of scroller
refitPlace (selection) item in view,by default the last element is placed in view
resizeResize scroller according to parent container only if parent container size increase
forceResizeResize scroller according to parent container.
updateUpdate list size according to parent container
repositionButtonRealign buttons with list


The plugin stores last visible item in data lastvisible when getLast/refit is called,first visible item in firstvisible and oldwidth and newwidth on resize.

Example 1.

This example has no "wrapper" div so the buttons are positioned with respect to body in this case. Try seeing the effect it has by resizing the window.This can be rectified by: 
			$('#test').scroller('repositionButton');
